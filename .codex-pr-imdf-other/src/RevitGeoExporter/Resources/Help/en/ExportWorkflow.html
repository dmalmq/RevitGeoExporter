Export Dialog

Export Workflow

The redesigned export dialog keeps the default path visible and moves technical settings behind expanders, so new users can finish a standard export without sorting through every advanced option first.

Export to Include in export Coordinate system Advanced options

Export to

  • The output folder is a required step and appears near the top of the dialog.
  • Use Browse... to select a new folder when needed.
  • Export stays disabled until you have at least one view, at least one feature type, and a destination folder.

Include in export

  • Choose the layers you want to export: Units, Details, Openings, and Levels.
  • The plain-language labels make the options easier to scan, while the exporter still writes the expected GeoPackage layers such as unit and opening.
  • Preview is available as soon as at least one plan view and one feature type are selected.

Coordinate system

  • The status card summarizes the current shared-coordinate context and resolved EPSG when available.
  • Expand Change coordinate settings only if you need to switch from shared coordinates to a converted target CRS or EPSG.
  • Expand View technical details when you need the longer model-coordinate breakdown for troubleshooting.

Advanced options

  • Advanced options start collapsed so the main export path stays focused.
  • Unit source only appears when Units is enabled, and the room category parameter only appears when the source is set to rooms.
  • Project mappings, diagnostics output, GIS package output, and legend-file options stay available without crowding the main workflow.

Before you commit files

The footer summarizes the current selection, and the exporter runs validation before writing files.

  • Use Preview and Assignments if you want to inspect geometry, warnings, or floor-category overrides first.
  • Use Validation and Diagnostics to understand what can block export or appear in the diagnostics report.
