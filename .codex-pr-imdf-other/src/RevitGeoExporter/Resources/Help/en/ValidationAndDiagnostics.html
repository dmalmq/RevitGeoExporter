Quality Checks

Validation and Diagnostics

Validation runs before files are written, so data issues can be caught early and reviewed while the export context is still open.

Validation checks include

  • Unassigned floor-derived units.
  • Duplicate or missing export IDs where required.
  • Empty or invalid geometry.
  • Unsupported opening families and unsnapped openings.
  • Views that produce no exportable output.
  • Missing vertical circulation output.

Diagnostics output

The diagnostics JSON can capture counts, warnings, skipped content, overrides, timestamps, and export timing. If GIS package output is enabled, the diagnostics report can travel with the handoff folder.

When export is blocked

Use the validation messages to decide whether you should return to preview, fix a settings issue, or adjust the export selection. The goal is to surface actionable issues before a partially understood package is delivered.
