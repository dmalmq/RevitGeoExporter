Configuration

Settings and Profiles

The settings hub separates stable global defaults from project-specific mappings, so teams can keep reusable settings without overwriting project decisions.

Global defaults

  • Default output folder and UI language.
  • Shared CRS / EPSG defaults.
  • Diagnostics and GIS package defaults.
  • Basemap source settings for preview.
  • Global export profiles.

Project data

  • Floor type to category mappings.
  • Family category overrides.
  • Accepted opening family names.
  • Project-scoped export profiles.

Import, export, and reset

Each scope can be imported, exported, or reset independently. If a saved file is incomplete or malformed, the status area explains what was ignored and safe defaults are used where possible.
