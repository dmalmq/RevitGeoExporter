Quality Checks

検証と診断

検証はファイルを書き出す前に実行されます。エクスポートの文脈が残っているうちに、データ上の問題や作業上の問題を見つけられます。

主な検証項目

  • カテゴリ未割り当ての床由来ユニット。
  • 必要なエクスポート ID の重複または欠落。
  • 空または無効なジオメトリ。
  • 未対応の開口ファミリや、スナップされていない開口。
  • 出力対象が存在しないビュー。
  • 縦動線フィーチャーの欠落。

診断出力

診断 JSON には件数、警告、スキップされた内容、上書き、タイムスタンプ、処理時間などが記録されます。GIS パッケージを有効にしている場合は、受け渡しフォルダーに同梱できます。

エクスポートが止まったとき

メッセージを手がかりに、プレビューへ戻るべきか、設定を見直すべきか、またはビュー選択を調整すべきかを判断します。中途半端な理解のまま成果物を渡さないためのチェックです。
